Complexity analysis

Solution 1:

We put n = length of arr

Time complexity: O(2^n)

Explanation:

A way to jump from the first cell to the last one is just a subset of arr, and an array of n elements has 2^n subsets, and we're traversing them recursively, the time complexity is O(2^n)

Space complexity: O(n)

Explanation:

We can have the case where we jump by one cell at a time and we have n cells, so the maximum call stack size is n+1

S(n) = n+1 = O(n)


Solution 2:

We put n = length of arr

Time complexity: O(n²)

Explanation:

We have nested for loops, the outer one is traversing elements of arr and the second one is traversing the cells that we can jump to starting from arr[i]

And the worst case is when we have an array like this: [5, 4, 3, 2, 1, 0], here the first iteration will cost 5 then the next one 4 and so on until 0

In a general way, the first iteration will cost n-1 then the next one n-2 and so until 0

T(n) = n+(n-1)+(n-2)+...+0 = n(n-1)/2 = (n²-n)/2 = (1/2n²)-(1/2n) = O(n²)

Space complexity: O(n)

Explanation:

We are using an array dp of n elements

S(n) = n = O(n)
