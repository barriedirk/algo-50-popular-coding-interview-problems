Complexity analysis

Solution 1:
We put |V| = number of vertices
              |E| = number of edges

Time complexity: O(|V|+|E|) (assuming that we're working with an adjacency list)

Explanation:

Each vertex is processed only once, and same thing for edges

In case of an undirected graph, edges are bidirectional, so they are processed twice

But by removing the constant, we get O(|V|+|E|) again, because |V|+2|E| is in O(|V|+|E|)

Space complexity: O(|V|)

Explanation:

The maximum call stack size is |V|
