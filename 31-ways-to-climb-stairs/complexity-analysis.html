Complexity analysis

Solution 1:

We put n = number of stairs

             m = length of possible steps

Time complexity: O(m^n)

Explanation:

We will use the recurrence relation method

In the base case, when n is equal to 0, we directly return, so T(0,m) = 1

In the recursive case, we traverse the m possible steps and we call the function at most m times with one step further (n-1), so T(n,m) = mT(n-1)+m

We keep replacing:

T(0,m) = 1

T(n,m) = mT(n-1,m) + m

T(n,m) = m(mT(n-2,m)+m)+m

T(n,m) = m^2 T(n-2,m)+m^2+m

T(n,m) = m^2 (mT(n-3,m)+m)+m^2+m

T(n,m) = m^3 T(n-3,m)+m^3+m^2+m

T(n,m) = m^3 (mT(n-4,m)+m)+m^3+m^2+m

T(n,m) = m^4 T(n-4,m)+m^4+m^3+m^2+m

T(n,m) = m^k T(n-k,m)+m^k+m^(k-1)+…+m^2+m

T(n,m) = m^n T(n-n,m)+m^n+m^(n-1)+…+m^2+m

T(n,m) = m^n T(0,m)+m^n+m^(n-1)+…+m^2+m

T(n,m) = m^n+ m^n+m^(n-1)+…+m^2+m

T(n,m) = m^n+ ?(m^k), k: 1 to n

T(n,m) = m^n + m/(m-1)(m^n-1)

for large values of m, m/(m-1) is approximately 1

T(n,m) = m^n + m^n - 1

T(n,m) = 2m^n - 1

T(n,m) = O(m^n)

Space complexity: O(n)

Explanation:

The maximum call stack size is n+1 because in the worst case, we go down by 1 step only at each level (n, n-1, ..., 1, 0)

S(n,m) = n+1 = O(n)


Solution 2:

We put n = number of stairs

             m = length of possible steps

Time complexity: O(n*m)

Explanation:

The outer loop is going from 1 to n+1, so it does n iterations, and the inner loop is traversing the possible steps, so it does m iterations, and we have O(1) operations inside

T(n,m) = n*m*O(1) = O(n*m)

Space complexity: O(n)

Explanation:

We are using an extra array of n+1 elements

S(n,m) = n+1 = O(n)
