Complexity analysis

Solution 1:

We put n = length of str

Time complexity: O(n.2^n)

Explanation:

We will use the recurrence relation method

In the base case, we join the characters of subsequences, which can have at most n characters, so

T(0) = n

In the recursive case, we recursively call the function twice starting from the next character, and remaining stuff is O(1), so

T(n) = 2T(n-1)+1

We keep replacing:

T(n) = 2(2T(n-2)+1)+1

T(n) = 4T(n-2) + 3

T(n) = 4(2T(n-3)+1)+3

T(n) = 8T(n-3) + 7

T(n) = 8(2T(n-4)+1)+7

T(n) = 16T(n-4) + 15

T(n) = 16(2T(n-5)+1)+15

T(n) = 32T(n-5) + 31

.

.

T(n) = (2^k)T(n-k) + (2^k)-1

.

.

T(n) = (2^n)T(n-n) + (2^n)-1

T(n) = (2^n)T(0) + (2^n)-1

T(n) = (2^n)n + (2^n)-1

T(n) = O(n.2^n)

Space complexity: O(n)

Explanation:

The maximum call stack size is n+1 but the subsequences array contains 2^n subsequences of at most n characters

S(n) = n+1+(n.2^n) = O(n.2^n)
