Solution 1:

Time complexity: O(n^5)

Explanation:

The explanation is detailed at the of the video

Space complexity: O(n)

Explanation:

The maximum call stack size is n+1 (n, n-1, ..., 0)

S(n) = n+1 = O(n)


Solution 2:

Time complexity: O(n)

Explanation:

The loop is doing n-1 iterations, and we're doing O(1) time work at each iteration

T(n) = (n-1)*O(1) = O(n)

Space complexity: O(n)

Explanation:

We're using a 2D array of n rows and 5 columns

S(n) = n*5 = 5n = O(n)


Space-optimized solution 2:

Time complexity doesn't change

Space complexity: O(1)

Instead of storing all the n rows, we work with 2 rows at a time only (prev and curr)

S(n) = 2*5 = 10 = O(1)


Solution 3:

Time complexity: O(1)

Explanation:

If we consider multiplication and division as O(1), then we're just multiplying then dividing, T(n) = O(1)

Space complexity: O(1)

Explanation:

We're not using input-size related extra space
