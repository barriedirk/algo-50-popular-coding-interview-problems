Complexity analysis

Solution 1:

We put n = the number of nodes in the linked list

Time complexity: O(n²)

Explanation:

The while loop traverses the nodes of the linked list, and each iteration costs O(1), the total is n

The outer for loop does n/2 iterations, the inner for loop does n-i-1 iterations, and what's inside it has an O(1) cost

We can represent the total cost with this sum:

Which is equal to 1/8(3n²-2n)

T(n) = n + 1/8(3n²-2n) = n + 3/8 n² - 1/4 n = 3/8 n² + 3/4 n = O(n²)
(the first n comes from the while loop)

Space complexity: O(1)

Explanation:

Our variables are: length, temp, left, i, j, and right

S(n) = 6 = O(1)


Solution 2:

We put n = the number of nodes in the linked list

Time complexity: O(n)

Explanation:

The reverseList() function has an O(n) cost where n is the number of nodes of the linked list we gonna reverse

The first loop does n/2 iterations because fast is jumping by 2 nodes at each iteration

slow = reverseList(slow) costs n/2 because we are reversing the right half of the linked list

Then we are comparing two linked lists of n/2 nodes (the left half and the right half), so the last while loop does n iterations, and each iteration has an O(1) cost

T(n) = n/2 + n/2 + n/2 = 3n/2 = O(n)

Space complexity: O(1)

Explanation:

We're just using two variables slow and fast, and reverseList() function has an O(1) space complexity, so:

S(n) = 1 + 1 + 1 = 3 = O(1)
